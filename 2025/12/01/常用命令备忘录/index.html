📂 基础操作

1. 进入博客目录

每次操作前，必须先在 Git Bash 中进入博客根目录：

2. 本地预览

改动后，在本地查看效果：

  • clean: 清除缓存（解决样式不更新的问题）。
  • s (server): 启动服务器。
  • 访问地址: http://localhost:4000
  • 停止预览: 在命令行按 Ctrl + C

3. 部署到GitHub Pages

  • g (generate): 生成静态网页文件。
  • d (deploy): 推送到 GitHub 仓库

📝 写作指令

1. 创建新post

  • 生成位置：source/_posts/文章标题.md

2.创建新页面

  • 生成位置： source/页面英文名/index.md
  • 用途: 创建如“About”、“Reviews”、“Projects”等非文章类页面。

3.草稿

  • 文件会生成在 source/_drafts。
  • 预览草稿: hexo s –draft
  • 发布草稿: hexo publish “草稿标题”
